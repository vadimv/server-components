# Property-Based Testing (PBT) Recommendations

This document outlines high-value opportunities for adding Property-Based Tests to the project. These tests focus on verifying invariants, round-trip properties, and algebraic consistency, which are difficult to cover exhaustively with example-based unit tests.

## 1. `rsp.server.Path` (High Priority)

**Why:** Fundamental value object used for routing and request handling. Logic is algebraic and self-contained.

**Properties:**
*   **Round-Trip Serialization:**
    `Path.of(p.toString()).equals(p)`
    *Verifies that parsing and string conversion are inverse operations.*
*   **Resolve/Relativize Consistency:**
    `base.resolve(rel).relativize(base).equals(rel)`
    *(Assuming `base` is absolute and `rel` is relative).*
*   **Absolute Resolution:**
    `base.resolve(absolutePath).equals(absolutePath)`
    *Verifies that resolving an absolute path ignores the base path.*
*   **Self-Containment:**
    `p.startsWith(p)` is always true.
*   **Empty Path Identity:**
    `p.resolve(Path.EMPTY).equals(p)`

## 2. `rsp.server.http.Query`

**Why:** Handles URL query string parsing and construction. Edge cases with special characters are common.

**Properties:**
*   **Round-Trip Serialization:**
    `Query.of(q.toString()).equals(q)`
    *Verifies encoding/decoding logic. Note: Current implementation might fail with special characters like `&` or `=`, identifying limitations or bugs.*
*   **Parameter Preservation:**
    `Query.of("key=" + value).parameterValue("key").equals(value)`
    *Verifies that values are preserved exactly.*

## 3. `rsp.util.json.JsonParser`

**Why:** Parsing logic is complex and prone to edge cases (escaped characters, nested structures).

**Properties:**
*   **Round-Trip:**
    `JsonParser.parse(JsonUtils.toString(data)).equals(data)`
    *Verifies that any valid JSON structure generated by the system can be parsed back to an equivalent object.*

## 4. `rsp.page.LivePageSession` (State Consistency)

**Why:** Core engine for state management and event handling.

**Properties:**
*   **State Consistency:**
    Apply a random sequence of valid events (clicks, inputs) to a session.
    *Invariant: The session should never throw an unexpected exception (500 error) and internal state maps (event handlers, refs) should remain consistent.*

## 5. `rsp.i18n.Translate`

**Why:** String formatting can fail with mismatched arguments.

**Properties:**
*   **Robustness:**
    `Translate.format(pattern, args)` should never throw an exception.
    *Even if `args` are missing or the pattern is malformed, it should return a safe fallback string.*
